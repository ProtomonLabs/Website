Over 90% of GWAS loci fall in non-coding regions. Interpreting them requires more than a single tool — it requires infrastructure that layers statistical, functional, and evolutionary data into a coherent analysis framework.

## The Infrastructure Challenge

Non-coding GWAS interpretation demands:

1. **Fine-mapping at scale** — statistically prioritizing causal variants across thousands of loci
2. **Regulatory data integration** — overlaying ATAC-seq, ChIP-seq, and Hi-C data from relevant tissues
3. **Functional scoring** — computing conservation, regulatory impact, and deleteriousness
4. **Variant-to-gene linking** — connecting non-coding variants to their target genes

No single tool solves all four. The solution is an **infrastructure stack** where each layer is independently maintained and composable.

## Our Infrastructure Stack

### Layer 1: Statistical Fine-mapping

```python
# Standardized fine-mapping pipeline
fine_mapping = FineMappingInfrastructure(
    method="susie",          # SuSiE, FINEMAP, CAVIAR
    ld_reference="gnomad",   # population-matched LD
    priors=annotated_variants
)
credible_sets = fine_mapping.run(summary_stats, loci)
```

Containerized, versioned, and reproducible. Swap methods without rebuilding the pipeline.

### Layer 2: Regulatory Data Platform

```
Tissue-specific regulatory data as a service:

  API: annotate.protomonlabs.org/v1/regulatory
  Query: /variants?tissue=heart&datasets=atac,chip,hic
  
  Returns: overlapping regulatory elements,
           chromatin interactions,
           transcription factor binding sites
```

Standardized API access to regulatory data means analysis pipelines stay clean while the underlying data sources evolve.

### Layer 3: Functional Score Integration

```python
# Composite scoring infrastructure
scores = ScoreIntegrator(
    conservation=["phyloP", "phastCons", "CADD"],
    regulatory=["gcms", "funseq2", "eigen"],
    tissue_specific=["ABC_score", "EpiMap"]
)
results = scores.compute(credible_set)
```

Each score source is independently versioned and cached. Adding a new score is a configuration change, not a code change.

### Layer 4: Variant-to-Gene Linking

```python
# Multi-evidence gene linking
links = GeneLinker(
    methods=["distance", "eQTL", "Hi-C", "ABC"]
)
target_genes = links.link(variants, tissue=heart)
```

Evidence-weighted gene assignment that's transparent and reproducible.

## Why Infrastructure Matters

Without infrastructure, non-coding GWAS analysis looks like this:
- Hand-curated scripts for each data source
- Manual downloads from different portals
- Ad-hoc versioning
- Results that can't be reproduced six months later

With infrastructure:
- One pipeline, any locus
- Versioned, cached data layers
- Reproducible by default
- Results that compound — every analysis improves the next
